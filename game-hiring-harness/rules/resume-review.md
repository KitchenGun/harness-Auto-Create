# resume-review — 이력서/포트폴리오 검토 체크리스트

## 발생일
2026-04-07

## 목적
게임 프로그래머 지원자의 이력서와 포트폴리오를 3인격이
각자의 관점에서 체계적으로 검토하기 위한 체크리스트.

---

## P1 이력서 검토 체크리스트 (HR 관점)

### 기본 정보 확인
- [ ] 연락처, 이름, 지원 직군 명시 여부
- [ ] 경력 기간 계산 및 공백 기간 확인
- [ ] 학력 및 관련 자격증 확인
- [ ] 희망 연봉 / 처우 조건 확인

### 경력 패턴 분석
- [ ] 이직 주기 확인 (1년 미만 반복 → 레드플래그)
- [ ] 프로젝트 완주율 (출시까지 간 프로젝트 비율)
- [ ] 역할 성장 추이 (주니어 → 시니어 → 리드 등)
- [ ] 게임 업계 경력 vs 타 업계 경력 비율

### 소프트 스킬 신호
- [ ] 자기소개서에서 "나" vs "우리" 사용 비율
- [ ] 성과 기술 시 구체적 수치 포함 여부
- [ ] 실패 경험 언급 여부 (성장 지표)
- [ ] 커뮤니티/오픈소스 기여 활동

### P1 서류 판정 기준
- PASS: 경력 일관성 + 성장 추이 + 레드플래그 없음
- HOLD: 경미한 우려 1-2개 (면접에서 확인 필요)
- FAIL: 레드플래그 2개 이상 또는 치명적 불일치

## P2 포트폴리오 검토 체크리스트 (기술 관점)

### 코드 품질 (포트폴리오/GitHub)
- [ ] 코드 가독성 (네이밍, 구조, 주석)
- [ ] 디자인 패턴 적절한 사용
- [ ] 에러 처리 및 방어적 프로그래밍
- [ ] 메모리 관리 패턴 (특히 C++)
- [ ] 테스트 코드 존재 여부

### 프로젝트 분석
- [ ] 독창적 구현 vs 튜토리얼 복사 구분
- [ ] 프로젝트 규모 및 복잡도
- [ ] 사용 엔진/프레임워크 숙련도 판단
- [ ] 빌드/배포 환경 구성 여부

### 기술 스택 평가
- [ ] 주력 언어 숙련도 (C++/C#/Python 등)
- [ ] 엔진 경험 (UE5/Unity/커스텀 엔진)
- [ ] 버전 관리 활용도 (Git 커밋 히스토리 분석)
- [ ] CI/CD 경험 여부
- [ ] 지원 분야 관련 기술 깊이

### 경력별 기대 수준

| 항목 | 주니어 | 미들 | 시니어 |
|------|--------|------|--------|
| GitHub 커밋 | 꾸준한 활동 | 의미 있는 기여 | 오픈소스 리딩 |
| 프로젝트 수 | 2-3개 (개인/팀) | 출시작 1+ | 출시작 3+ |
| 코드 품질 | 기본 규칙 준수 | 패턴 활용 | 아키텍처 설계 |
| 문서화 | README 존재 | 기술 문서 | 설계 문서 |

### P2 서류 판정 기준
- PASS: 지원 직급 대비 기술 수준 충족 + 독창적 구현 확인
- HOLD: 기본기 있으나 깊이 부족 (코딩 테스트로 추가 확인)
- FAIL: 튜토리얼 수준 + 기본기 미달 + 레드플래그

## P3 종합 서류 검토

### 교차 검증 항목
- [ ] 이력서 기술 스택 vs 포트폴리오 실제 사용 기술 일치 여부
- [ ] 경력 기술 내용 vs 코드 수준 일치 여부
- [ ] "리드 경험" 주장 vs 실제 코드 리뷰/멘토링 흔적
- [ ] 자기소개서 기술 역량 vs 포트폴리오 증거

### P3 최종 서류 판정
```
[P3 서류 종합 판정]
- P1 판정: PASS / HOLD / FAIL
- P2 판정: PASS / HOLD / FAIL
- 교차 검증: 일치 / 불일치 (상세)
- 최종: 면접 진행 / 추가 자료 요청 / 불합격
- 특이사항: ...
```
