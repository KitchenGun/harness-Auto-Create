# AGENTS.md — 하네스 프로젝트 컨텍스트

## 원칙
- 처음부터 완벽하지 않아도 된다. 실패할 때마다 한 줄씩 여기에 추가한다.
- 성공은 조용히, 실패만 시끄럽게.
- 모든 규칙은 실제 실패 경험에서 나온다.

## 코드 규칙
- 커밋 전 반드시 `./scripts/lint.sh` 통과할 것.
- 린터 빨간불 → 자동 교정 시도 → 실패 시에만 사람에게 알림.
- 4000줄 통과 결과는 출력하지 않는다. 실패한 줄만 보여준다.
- 사용하지 않는 코드(dead code)는 발견 즉시 삭제한다.

## 문서 규칙
- 문서는 코드와 동기화되어야 한다. 코드가 바뀌면 문서도 바뀐다.
- 이 파일은 60줄을 넘기지 않는다. 세부사항은 `rules/` 디렉토리에 둔다.

## 자동화 흐름
1. 코드 변경
2. pre-commit 훅 → lint + 가비지 컬렉션
3. 실패 → 자동 교정 루프 (최대 3회)
4. 여전히 실패 → 에러 리포트 + rules/ 에 새 규칙 후보 기록
5. 통과 → 조용히 커밋

## 가비지 컬렉션 체크리스트
- [ ] 문서 ↔ 코드 불일치 없는가?
- [ ] 규칙을 위반한 코드가 있는가?
- [ ] 사용하지 않는 import/변수/함수가 있는가?
- [ ] 실패 → lint 새 규칙 추가 → 하네스 진화
