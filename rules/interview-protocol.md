# interview-protocol — 3인격 면접 질문 생성 및 진행 규칙

## 발생일
2026-04-07

## 목적
게임 프로그래머 면접에서 P1(HR), P2(개발), P3(통합)가
각자의 관점에서 질문을 생성하고 합의 기반으로 면접을 진행하는 프로토콜.

---

## 면접 단계

### 1단계: 서류 기반 질문 생성 (면접 전)

#### P1이 생성하는 질문 (인성/조직적합성)
- "가장 힘들었던 팀 갈등 상황과 해결 과정을 말씀해 주세요."
- "크런치 타임에 대한 본인의 생각과 경험은?"
- "비개발 직군(기획, 아트)과의 의견 충돌 경험은?"
- "왜 우리 회사/프로젝트에 지원했나요?"
- "3년 후 커리어 목표는?"

#### P2가 생성하는 질문 (기술)
- **코드 리뷰 질문**: 지원자 포트폴리오 코드에서 특정 부분을 지목하여 설계 의도 질문
- **아키텍처 질문**: "이 시스템을 1000명 동접으로 확장한다면 어떻게 변경하겠습니까?"
- **디버깅 시나리오**: "플레이어가 특정 조건에서만 벽을 뚫는 버그가 보고됐습니다. 어떻게 추적하시겠습니까?"
- **최적화 질문**: "프레임 드랍이 발생합니다. 원인 파악 절차를 설명해 주세요."
- **도메인 질문**: 지원 분야(클라이언트/서버/엔진)에 맞는 심화 질문

#### P3가 생성하는 질문 (교차 검증)
- P1 질문의 답변에서 기술적 일관성 확인
- P2 질문의 답변에서 커뮤니케이션 능력 확인
- "가장 자랑스러운 기술적 성취를 비개발자에게 설명하듯 말씀해 주세요."

---

### 2단계: 면접 진행 규칙

| 규칙 | 설명 |
|------|------|
| 질문 비율 | P1 질문 30% + P2 질문 50% + P3 질문 20% |
| 시간 배분 | 총 60분 기준: 인성 15분 + 기술 30분 + 교차검증 10분 + Q&A 5분 |
| 후속 질문 | 답변이 모호하면 최대 2회까지 후속 질문 허용 |
| 평가 시점 | 각 질문 직후 P1·P2가 독립 채점 (5점 척도) |
| 합의 시점 | 면접 종료 후 P3가 종합 판정 |

---

### 3단계: 면접 후 평가

#### 채점 기준 (질문별 5점 척도)
- 5점: 기대 초과 — 깊이 있는 인사이트 + 구체적 경험
- 4점: 기대 충족 — 정확한 답변 + 실무 경험 뒷받침
- 3점: 기본 수준 — 교과서적 답변, 실무 경험 부족
- 2점: 미흡 — 핵심을 놓침, 우회적 답변
- 1점: 부적합 — 질문 이해 못 함, 관련 없는 답변

---

## 경력별 면접 초점

### 주니어 (0-3년)
- P1 초점: 학습 의지, 피드백 수용도, 팀워크
- P2 초점: CS 기초, 코딩 테스트, 엔진 기본 조작
- P3 초점: 성장 잠재력 vs 즉시 전력 감의 균형

### 미들 (3-7년)
- P1 초점: 리더십 잠재력, 멘토링 의지, 프로젝트 관리
- P2 초점: 아키텍처 설계, 최적화 경험, 라이브 서비스
- P3 초점: 기술 깊이 vs 팀 기여도의 균형

### 시니어 (7년+)
- P1 초점: 조직 영향력, 비전, 팀 빌딩
- P2 초점: 시스템 설계, 기술 의사결정, 장애 대응
- P3 초점: 기술 리더십 vs 관리 역량의 균형
